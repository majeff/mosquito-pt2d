# 硬體連接詳細說明

## 🔌 組件清單

### 必需組件

1. **Arduino 開發板**
   - Arduino Uno / Nano / Mega 2560
   - **推薦使用 Mega 2560**（有多個硬體串口，無需軟串口）

2. **總線舵機（串口舵機）**
   - 型號: LewanSoul LX-16A, Feetech SCS15, HTS等
   - 數量: 2 個
   - 規格:
     - 通訊方式: 串口 (UART)
     - 工作電壓: 6V-8.4V
     - 波特率: 9600 或 115200
     - 轉動角度: 0-240°
     - 控制方式: 位置、速度、扭矩
     - 可串聯多個舵機（總線）

3. **2D 雲台支架**
   - 材質: 鋁合金或塑膠
   - 包含兩個舵機安裝位
   - Pan (水平) 和 Tilt (垂直) 軸

4. **電源供應**
   - 6V-8.4V / 2A 以上直流電源（推薦 7.4V 鋰電池）
   - 或使用 12V 電源 + DC-DC 降壓模組

5. **其他**
   - 杜邦線若干
   - USB 線（上傳程序、上位機通訊用）

### 總線舵機 vs 普通舵機

| 特性 | 總線舵機 | 普通 PWM 舵機 |
|------|---------|--------------|
| 控制方式 | 串口 (UART) | PWM 信號 |
| 連接方式 | 可串聯（總線） | 每個需獨立引腳 |
| 反饋 | 可讀取位置/溫度等 | 無反饋 |
| 精度 | 高（0.24° 或更高） | 中（~1°） |
| 引腳占用 | 2個（TX/RX） | 每個舵機 1 個 PWM |
| 設置 ID | 可修改 ID | 無 |
| 價格 | 較高 | 較低 |

## 📐 引腳連接表

### Arduino Mega 2560（推薦）

| Arduino 引腳 | 功能 | 連接到 | 備註 |
|-------------|------|--------|------|
| TX1 (Pin 18) | 舵機總線 TX | 舵機 RX 線 | 黃線 |
| RX1 (Pin 19) | 舵機總線 RX | 舵機 TX 線 | 綠線 |
| USB (Serial) | 上位機通訊 | 電腦 USB | 或使用 TX0/RX0 |
| 外接電源 | 舵機供電 | 舵機 VCC | 6V-8.4V |
| GND | 接地 | 舵機 GND 和電源 GND | 共地 |

### Arduino Uno / Nano

| Arduino 引腳 | 功能 | 連接到 | 備註 |
|-------------|------|--------|------|
| Pin 10 | 舵機總線 TX (軟串口) | 舵機 RX 線 | 黃線 |
| Pin 11 | 舵機總線 RX (軟串口) | 舵機 TX 線 | 綠線 |
| USB (Serial) | 上位機通訊 | 電腦 USB | 占用 TX0/RX0 |
| 外接電源 | 舵機供電 | 舵機 VCC | 6V-8.4V |
| GND | 接地 | 舵機 GND 和電源 GND | 共地 |

## 🔧 詳細連接步驟

### 步驟 1: 總線舵機接線

每個總線舵機有 3 根線：

| 線材顏色 | 功能 | 連接到 |
|---------|------|--------|
| 紅色 | VCC (電源+) | 外接 6V-8.4V 電源正極 |
| 黑色 | GND (接地) | 外接電源負極和 Arduino GND |
| 黃色/綠色 | Signal (串口數據) | 總線（所有舵機串聯） |

**總線舵機串聯連接示意**:

```
                     總線數據線（黃/綠）
Arduino TX1 ──┬─────┬─────┬─────
              │     │     │
           舵機1  舵機2  舵機3 ...
           (ID=1)(ID=2)(ID=3)
              │     │     │
Arduino RX1 ──┴─────┴─────┴─────

電源連接：
外接 7.4V ─┬── 所有舵機 VCC (紅線)
           └── Arduino VIN (可選)
```

#### 方案 B: 使用 LM7805 穩壓模組

```
7-12V 電源
    ↓
LM7805 穩壓模組
    ├── 5V 輸出 → 伺服馬達 VCC
    └── GND → 伺服馬達 GND + Arduino GND
```

#### 方案 C: 使用 Buck 降壓模組

```
12V 電源
    ↓
Buck 降壓模組 (調整到 5V)
    ├── 5V 輸出 → 伺服馬達 VCC
    └── GND → 伺服馬達 GND + Arduino GND
```

### 步驟 3: 串口連接

#### 與電腦連接（USB）

直接使用 USB 線連接 Arduino 和電腦，無需額外接線。

#### 與其他微控制器連接（UART）

```
Arduino TX (Pin 1) ──→ 對方 RX
Arduino RX (Pin 0) ←── 對方 TX
Arduino GND ───────── 對方 GND
```

⚠️ **注意**: 上傳程序時需斷開 TX/RX 連接，否則可能上傳失敗。

### 步驟 4: 添加濾波電容（可選但建議）

在電源端並聯電容以穩定電壓：

```
伺服馬達 VCC ──┬── 電容正極
               │
伺服馬達 GND ──┴── 電容負極

建議使用: 470μF 或 1000μF 電解電容
```

## ⚠️ 注意事項和常見問題

### 1. 電源問題

**問題**: 伺服馬達抖動、Arduino 重啟、USB 斷開

**原因**:
- Arduino USB 供電最大電流約 500mA
- 單個 SG90 伺服馬達靜態約 100mA，負載時可達 500-800mA
- 兩個伺服馬達同時工作可能導致電流不足

**解決方案**:
1. 使用外接 5V/2A 以上電源
2. 添加大容量電解電容（470μF-1000μF）
3. 將 Arduino 和伺服馬達分開供電

### 2. 共地問題

**問題**: 伺服馬達不工作或工作不正常

**原因**: Arduino 和外接電源沒有共地

**解決方案**:
- 確保 Arduino GND 和外接電源 GND 連接在一起
- 所有地線必須連接到同一個公共地點

### 3. 信號線干擾

**問題**: 伺服馬達抖動、位置不準確

**解決方案**:
- 信號線盡量短
- 遠離電源線
- 使用雙絞線或屏蔽線
- 添加 1kΩ 電阻串聯在信號線上

### 4. 上傳失敗

**問題**: Arduino IDE 或 PlatformIO 上傳程序失敗

**原因**: TX/RX 引腳被占用

**解決方案**:
- 上傳前斷開 TX/RX 連接
- 上傳完成後重新連接
- 或使用 Arduino Mega 的 Serial1 (TX1/RX1)

### 5. 過熱問題

**問題**: 伺服馬達或穩壓模組發熱

**解決方案**:
- 檢查電源電壓是否過高
- 添加散熱片
- 避免伺服馬達長時間堵轉
- 使用更大功率的穩壓模組

## 🧪 測試步驟

### 1. 基本測試

上傳程序後：

1. 打開串口監視器（115200 波特率）
2. 觀察啟動信息
3. 伺服馬達應移動到初始位置（90°, 90°）

### 2. 功能測試

```bash
# 1. 測試絕對移動
<MOVE:90,90>    # 應該回到中心位置
<MOVE:0,0>      # 移動到最小角度
<MOVE:180,180>  # 移動到最大角度

# 2. 測試相對移動
<MOVER:10,0>    # Pan 軸增加 10°
<MOVER:-10,0>   # Pan 軸減少 10°
<MOVER:0,10>    # Tilt 軸增加 10°

# 3. 測試速度控制
<SPEED:10>      # 設置慢速
<MOVE:180,90>   # 觀察緩慢移動
<SPEED:100>     # 設置快速
<MOVE:0,90>     # 觀察快速移動

# 4. 測試其他功能
<POS>           # 獲取當前位置
<HOME>          # 回到初始位置
<CAL>           # 執行校準
```

### 3. 電源測試

使用萬用表測量：
- 伺服馬達 VCC: 應為 4.8V-5.2V
- 負載時電壓降: 不應低於 4.5V
- Arduino 5V 引腳: 約 5V
